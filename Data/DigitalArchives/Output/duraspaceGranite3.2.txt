
Summary:
The document outlines various requirements for repository managers and users in a Digital Asset Management System (DAMS). The requirements include access to collections and objects, updating repository information, restricting curatorial access, listing collections and objects, contacting repositories, browsing formal collections, describing collections, indicating names related to collections, subjects, and other aspects, knowing events occurring to collections, browsing by collections, limiting search, knowing components belonging to an object, knowing files belonging to an object, representing the physical and/or logical structure of an object, listing components in their appropriate order, linking components or files, describing each component, indicating names and subjects pertinent to components, indicating copyright status and statutes for components, knowing events occurring to components over their life cycle, presenting components comprising an object in assigned order, seeing and navigating the component hierarchy of an object, knowing content files in the DAMS for repository's objects, associating a content file with either an object or an object component, knowing any event that occurred to a content file for any of repository's objects, indicating aspects of a master file useful for monitoring its integrity and impending obsolescence, providing a title for repository's collections, objects, and object components, knowing changes occurring to titles for repository's collections, objects, and object components, presenting collections, objects, and object components by title, knowing the place of publication, publisher, edition, etc., when pertinent, describing various facets of a collection, object, or object component that are not addressable via title, date, name, and subject entities, knowing modifications made to notes for repository's collections, objects, or object components, knowing languages for presenting in a collection, object, or object component, knowing changes made to language(s) indicated for a collection, object, or object component, knowing the geographical context of a collection, object, or object component, where pertinent, indicating all subjects that are pertinent to a collection, object, or object component, knowing other pertinent versions or descriptions of a resource that are external to the DAMS, knowing if the checksum for any of repository's content files has changed since last registered, knowing hardware and software used to generate a content file, knowing what objects or object components are still under copyright and cannot be displayed without either a license or incurring risk, knowing when copyright for an object or object component is likely to expire per jurisdictional copyright statute, knowing when the pertinence of a statute to an object or object component is scheduled to expire, knowing if copyrighted content has an applicable license, knowing if an object or object component is subject to a library policy that restricts access in any way, knowing when application of a library policy to an object or object component is scheduled to expire, knowing if application of a library policy to an object or object component has been modified, either manually or automatically, knowing all uses for an object or object component that are permitted or restricted per the rights status of the object or object component, knowing the dates for when a permission or restriction is active, knowing when permitted and restricted acts and/or their active dates are modified, tracking all events scheduled for recording in the DAMS, knowing the agent(s) responsible for carrying out an event, knowing when the event occurred, knowing the outcome of the event, e.g., object NN was deleted.
